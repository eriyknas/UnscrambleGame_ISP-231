# Лабораторная работа №13-14. ViewModel и архитектура Android-приложений
## Описание игры
Игра-головоломка со словами, где пользователю показывается слово с перемешанными буквами, а он должен ввести правильный вариант.
## Технологии
- Kotlin
- Jetpack Compose 
- ViewModel
- StateFlow
## Функциональность приложения
- игровая логика
- пользовательский интерфейс
- жизненный цикл 
- управление состоянием
- тестирование
## Контрольные вопросы
1. ViewModel — это специальный класс из Android Jetpack.
- Он решает проблему потери данных при изменении конфигурации. 
- Без ViewModel при повороте экрана пересоздается, и все переменные в ней сбрасываются.
2. MutableStateFlow - это изменяемый поток данных. Он имеет методы для обновление своего значения. StateFlow - это неизменяемая версия, доступная только для чтения.
- _uiState — приватная переменная. Это изменяемое состояние
(MutableStateFlow), которое можем менять только внутри ViewModel.
- uiState — публичная переменная. Это неизменяемое представление
(StateFlow), которое UI может только читать. 
- Backing property pattern - это техника, когда мы обновляем приватное изменяемое поле и публичное неизменяемое поле для него.
3. Хранение Context в ViewModel может привести к утечке памяти. 
- Если ViewModel хранит ссылку на Activity, сборщик мусора не сможет уничтожить Activity после ее закрытия, так как на нее все еще есть ссылка.
4.  StateFlow живет в ViewModel и предназначен для хранения бизнес-логики и состояния, которое должно сохраняться при изменении конфигурации.
- remember - просто, но ненадёжно, данные хранятся в Composamle, используется для локального состояния UI, которое не важно сохранять
- rememberSaveable - ограничен простыми типами, данные хранятся в Bundle, используется для локального состояния UI, которое нужно сохранить при повороте
5.  data class — специальный тип класса в Kotlin, который автоматически создаёт методы equals(), hashCode(), toString(), copy(). Идеально подходит для хранения данных. Используется для состояния UI позволяет удобно управлять неизменяемым состояниям.
- функция copy() позволяет создать новую копию объекта с измененными полями
6. CollectAsState() - это функция, которая подписывается на StateFlow внутри Composable.
- Когда заначение StateFlow меняется, collectAsState() автоматически триггерит рекомпозицию тех частей UI.
7. Это принцип разделения ответственности. Пакет data содержит модели данных и бизнес-логику. Пакет ui содержит все, что связано с отображением.
- Соответствует принципам чистой архитектуры.
8. Когда приложение полностью закрывается или ОС убивает процесс для освобождения памяти, ViewModel уничтожается и теряет все данные.
- SavedStateHandle позволяет ViewModel выживать при process death. DataStore для хранения простых настроек и данных. Room Database для структурированных данных и сложных запросов. WorkManager для отложенных фоновых задач.
